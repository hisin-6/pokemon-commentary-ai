# OCRテスト用画像

Sprint 1のEasyOCR動作確認に使う画像置き場。

## 推奨スペック

| 項目 | 推奨 |
|------|------|
| 形式 | **PNG**（圧縮による劣化なし） |
| 解像度 | キャプチャそのままのサイズ（1920×1080 など） |
| 色 | カラーでOK（OCRが内部でグレースケール変換する） |

## 集めてほしい場面（優先度順）

### 優先度: 高（Sprint 1 完了基準に直結）

| ファイル名例 | 場面 | 確認したいテキスト |
|------------|------|----------------|
| `turn_start.png` | ターン開始・両ポケモン表示 | ポケモン名・HP数値 |
| `move_select.png` | 技選択画面（4つの技が表示） | 技名 |
| `hp_low.png` | HP が赤くなっている状態 | HP数値・ポケモン名 |

### 優先度: 中（精度チューニング用）

| ファイル名例 | 場面 | 確認したいテキスト |
|------------|------|----------------|
| `switch_select.png` | 交代選択画面 | 控えのポケモン名・HP |
| `result_win.png` | 勝利画面 | 結果テキスト |
| `result_lose.png` | 敗北画面 | 結果テキスト |
| `weather_rain.png` | あめ・すなあらし等の天気発動中 | 天気テキスト |

### 優先度: 低（Sprint 5 の辞書補正チューニング用）

| ファイル名例 | 場面 | 備考 |
|------------|------|------|
| `tera_activate.png` | テラスタル発動シーン | テキストが派手なエフェクトで見えにくい |
| `damage_text.png` | ダメージ数値が表示されている瞬間 | 数値が動いていて認識しにくい場合がある |

## 撮り方のコツ

- **技エフェクト中は避ける** → テキストが隠れてOCR精度が下がる
- **UI が完全に表示されたタイミング** で撮る（フェードイン途中は不可）
- Windows の `Win + Shift + S` または `PrintScreen` でキャプチャ → PNG で保存

## 使い方

```bash
# WSL2から（venv有効化済み）
python src/capture/screen_capture.py --image data/test_images/ocr/turn_start.png
```
