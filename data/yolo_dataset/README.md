# YOLOv8 学習データセット

Sprint 4 でYOLOv8による状態異常・ボール数検出を実装するための学習データ置き場。
Sprint 1〜3 の間に少しずつ画像を集めておくと Sprint 4 がスムーズに進む。

## 検出対象クラス（予定）

| クラスID | クラス名 | 説明 |
|---------|---------|------|
| 0 | `status_burn` | やけど（炎アイコン） |
| 1 | `status_poison` | どく |
| 2 | `status_bad_poison` | もうどく |
| 3 | `status_paralysis` | まひ |
| 4 | `status_sleep` | ねむり |
| 5 | `status_freeze` | こおり |
| 6 | `ball_alive` | ボール（残りポケモン）- 生存 |
| 7 | `ball_fainted` | ボール（残りポケモン）- 気絶済み |

## ディレクトリ構成

```
yolo_dataset/
├── images/
│   ├── train/   ← 学習用画像（全体の80%）
│   └── val/     ← 検証用画像（全体の20%）
└── labels/
    ├── train/   ← 学習用アノテーション（画像と同名の.txtファイル）
    └── val/     ← 検証用アノテーション
```

## 推奨スペック

| 項目 | 推奨 |
|------|------|
| 形式 | **PNG または JPG**（YOLOはどちらも対応） |
| 解像度 | 元画像そのまま（リサイズ不要・YOLOが自動調整） |
| 枚数目安 | 各クラス最低50枚・できれば100〜200枚 |

## アノテーション形式（YOLO形式）

`labels/train/` に画像と同名の `.txt` ファイルを置く。

```
# labels/train/battle_001.txt の中身
# <クラスID> <x中心> <y中心> <幅> <高さ>（すべて0〜1の相対値）
0 0.312 0.423 0.045 0.032
6 0.820 0.150 0.018 0.025
```

## アノテーションツール（おすすめ）

- **[labelImg](https://github.com/HumanSignal/labelImg)**: 無料・シンプル・YOLO形式に直接エクスポートできる
  ```bash
  pip install labelImg
  labelImg
  ```

- **[Roboflow](https://roboflow.com/)**: ブラウザで使える・無料枠あり・チーム共有もできる

## 集める画像の優先度

Sprint 4 までに余裕があれば集めておく。

| 優先度 | 場面 | 理由 |
|--------|------|------|
| 高 | 状態異常が画面に表示されているターン | メインの検出対象 |
| 高 | ボール数が変化した直後（気絶後） | カウントに直結 |
| 中 | 状態異常なし（ネガティブサンプル） | 誤検知を減らすため |
| 低 | 激しいエフェクト中 | 難しいケースの学習用 |
