# ADR-003: 実況文生成にローカルLLM（Phi-3 mini 4bit量子化）を使用する

## ステータス
承認済み

## 日付
2026-02-24

## 文脈

実況文の生成は毎ターン必要であり、APIを毎回呼び出すとコストが積み重なる。一方でローカルLLMはVRAMを消費するため、3Dモデルの描画（4〜5GB）と共存できるモデルを選ぶ必要がある。

## 決定

Phi-3 mini（4bit量子化）をOllamaでローカル実行し、実況文を生成する。

## VRAM配分

| コンポーネント | 消費VRAM |
|--------------|---------|
| Phi-3 mini 4bit量子化 | 約2〜3 GB |
| 3Dモデル描画 | 約4〜5 GB |
| YOLO・OCR | 約0.5〜1 GB |
| 余裕 | 約1〜3 GB |
| **合計** | **10GB以内** |

## 理由

- 4bit量子化によりVRAM消費を抑えながら十分な文章生成品質を確保できる
- Ollamaにより簡単にローカル実行できる
- 実況文生成のみであればPhi-3 miniクラスで実用的な品質が出る
- 毎ターン呼び出してもAPIコストが発生しない

## 却下した選択肢

| 選択肢 | 却下理由 |
|--------|---------|
| Llama 3.1 8B（量子化なし） | VRAM 5〜6GB消費で3Dモデルと共存が厳しい |
| 毎回Bedrock APIで生成 | ターン数が多くコスト過大になる |
| Gemma 2 9B | Phi-3 miniより消費VRAMが大きい |

## 再検討条件

実況の品質がユーザー要件を満たさない場合は、以下を検討する。

- Llama 3.1 8B 4bit量子化に切り替える（VRAM約4〜5GB）
- 実況文生成もBedrock Claude Haikuに移行する

## 結果

RTX 3080（VRAM 10GB）の範囲内で3Dモデルと安定して共存できる。毎ターンのAPI呼び出しコストをゼロに抑えられる。
