# ADR-005: AWSはBedrock・EC2・S3を使用する

## ステータス
承認済み

## 日付
2026-02-24

## 文脈

開発者はすでにEC2とS3を利用している。AIサービスの追加とAPIキーの安全な管理、ログの保存先が必要。

## 決定

以下の役割分担でAWSを利用する。

| サービス | 役割 |
|---------|------|
| AWS Bedrock | Claude Haiku Vision・テキスト生成のAPI呼び出し |
| EC2 | APIキー管理・Bedrockへの中継サーバー（Flask） |
| S3 | 実況ログ・スクリーンショットの保存 |

## アーキテクチャ

```
ローカルPC
　↓ 画像・状況テキストを送信
EC2（Flask）
　↓ Bedrockを呼び出す
AWS Bedrock（Claude Haiku）
　↓ 実況テキストを返却
ローカルPC（VOICEVOX・3Dモデル）
　↓
S3（ログ・画像を保存）
```

## 理由

- EC2にAPIキーを置くことでローカルPCにキーを持たずセキュリティが向上する
- 既存のEC2・S3の知識をそのまま活用できる
- BedrockによりAWSの請求に一本化でき管理が容易
- LambdaなどよりEC2の方が学習コストが低く既存環境と親和性が高い

## 却下した選択肢

| 選択肢 | 却下理由 |
|--------|---------|
| Lambda（中継サーバー） | 学習コストあり・初期段階では不要 |
| Rekognition | ゲーム画面への適合性が不明 |
| ローカルからBedrock直接呼び出し | APIキーをローカルに置くリスクがある |

## 注意事項

- **AWSの請求アラートを最初に設定すること**（コスト超過防止）
- EC2インスタンスは常時起動にせず、開発・実況時のみ起動する運用を推奨

## 結果

既存のAWS環境を活用しながら安全にAIサービスを追加できる。Sprint 2でEC2サーバーの構築とBedrock連携を実装する。
