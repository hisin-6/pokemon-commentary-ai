# ポケモン対戦実況AI - Claude Code 引継ぎプロンプト

## プロジェクト概要

ポケモン対戦画面をリアルタイムで認識し、3Dモデルが音声で実況するAIシステムを開発する。

---

## 開発環境

- **OS**: Windows
- **GPU**: NVIDIA GeForce RTX 3080 / VRAM 10GB
- **言語**: Python
- **クラウド**: AWS（EC2・S3・Bedrock）
- **AWSの既存利用**: EC2とS3のみ（Bedrockは新規追加予定）

---

## システム構成（確定済み）

```
ゲーム画面キャプチャ（mss、1秒間隔）
　↓
OpenCV → 差分検出でターン切替・イベントを検知（ローカル）
EasyOCR → HP・ポケモン名・技名を取得（ローカル）
YOLOv8 → 状態異常・ボール数を検出（ローカル）
　↓ 大きな状況変化イベント時のみ（ターン切替・交代・気絶）
AWS Bedrock（Claude Haiku Vision）→ 画面全体の状況理解（Vision分析のみ）
　↓ 全イベント時
Ollama + Phi-3 mini 4bit量子化（ローカル）→ 実況文生成
　↓
VOICEVOX（ローカル）→ 音声読み上げ
　↓
3Dモデル（VRoid + バーチャルモーションキャプチャー）→ 口パク・表示（VB-Audio Virtual Cable 経由リップシンク）
　↓
AWS S3 → 実況ログ・スクリーンショット保存
```

> **注**: Bedrockは「画面全体の状況理解（Vision分析）」専用。実況文の生成はローカルのPhi-3 mini（ADR-003）が担当。（ADR-001改訂済み）

---

## アーキテクチャ決定記録（ADR）サマリー

| ADR | 決定内容 | 理由 |
|-----|---------|------|
| ADR-001 | Vision分析はAWS Bedrock Claude Haiku（状況理解専用） | VRAM節約・低コスト・AWS一本化 |
| ADR-002 | 画面認識はOCR＋YOLOのハイブリッド | APIコスト削減・ローカル処理で十分 |
| ADR-003 | 実況文生成はPhi-3 mini 4bit量子化（ローカル・Ollama） | VRAM 2〜3GBで3Dモデルと共存可能 |
| ADR-004 | 音声合成はVOICEVOX | 無料・高品質・CPU動作・GPU不要 |
| ADR-005 | AWSはBedrock＋EC2＋S3を使用 | 既存環境を活用・APIキーをEC2で管理 |
| ADR-006 | 3DモデルはVRoid Studio + バーチャルモーションキャプチャー + VB-Audio Virtual Cable | 無料・VRM 0.x 対応・音声自動リップシンク確認済（Sprint 3） |
| ADR-007 | イベント駆動アーキテクチャ（差分検知でトリガー制御） | API呼び出し削減・3秒以内の遅延目標 |

### VRAM配分（合計10GB以内）
- Phi-3 mini 4bit量子化: 約2〜3GB
- 3Dモデル描画: 約4〜5GB
- YOLO・OCR: 約0.5〜1GB
- 余裕: 約1〜3GB

## アジャイル開発計画

### Sprint 1（現在）: 画面キャプチャ＋OCR基本動作

**目標**: ポケモン対戦画面からHP・ポケモン名をテキストで取得できる状態にする

**タスク**:
1. `mss` で1秒ごとに画面キャプチャを取得する
2. `EasyOCR` でHP数値・ポケモン名・技名を取得する
3. `OpenCV cv2.absdiff` で差分検出しターン切替を検知する
4. 取得した情報をコンソールにログ出力する

**完了基準**:
- HP・ポケモン名がテキストとして取得できる
- ターン切替のタイミングでログが出力される

---

### Sprint 2: AWS Bedrock接続＋実況文生成

**タスク**:
1. EC2にFlaskサーバーを立て、APIキーをEC2で管理する
2. AWS BedrockからClaude Haikuを呼び出す
3. ポケモン専門用語をシステムプロンプトに組み込む（タイプ相性・技・型名）
4. イベント時のみAPIを呼ぶ差分トリガーを実装する
5. 実況ログをS3に保存する

---

### Sprint 3: VOICEVOX＋3Dモデル連携

**タスク**:
1. VOICEVOXをセットアップし実況テキストを音声再生する
2. 3DモデルをVRoid等でセットアップし表示する
3. 音声と口パクを同期させる

**完了状況**（Sprint 3 完了）:
- VOICEVOX インストール済み（Windows・ localhost:50021）
- バーチャルモーションキャプチャーで VRM 0.x モデル表示確認済み
- VB-Audio Virtual Cable 経由で VOICEVOX 音声連動リップシンク確認済み
- 残り: `voicevox_client.py` / `audio_player.py` 実装（Python 自動化）
- TODO: VRM 1.0 モデルを VRM 0.x に変換・正式モデルに差し替え

---

### Sprint 4: YOLOv8導入＋全体パイプライン統合

**タスク**:
1. YOLOv8で状態異常アイコン・ボール数を検出する
2. 全コンポーネントをつなぎ1試合を通して実況が流れる状態にする
3. API呼び出しコストを計測し1試合15円以内に収まることを確認する

---

### Sprint 5: 品質改善＋ポケモン知識強化

**タスク**:
1. 直前の実況履歴をプロンプトに含め文脈を保持する
2. 専門用語辞書を充実させる（HBサーフゴー・ミラー・テラスタル等）
3. イベント検知から音声再生まで3秒以内を目標に遅延改善する

---

## 注意事項・リスク

- VRAM不足になった場合はLLMをさらに量子化するか処理をEC2に逃がす
- OCRがポケモン名を誤認識した場合はポケモン名辞書で後補正する
- ゲームのUIが更新されると再学習が必要になる可能性がある

---

## まず最初にやること（Sprint 1）

```bash
pip install mss easyocr opencv-python
```

上記ライブラリをインストールし、ポケモン対戦画面のスクリーンショットを取得して
EasyOCRでHP・ポケモン名が正しく読み取れるか確認するスクリプトを作成してください。
